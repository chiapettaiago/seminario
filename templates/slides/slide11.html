Active Voice

Present:
The developer writes the code.

Past:
The team created the software.

Future:
They will deploy the app.

Passive Voice

Present:
The code is written by the developer.

Past:
The software was created by the team.

Future:
The app will be deployed.

## Passive Voice Structure
# Structure: Subject + to be + Past Participle (+ by agent) Active: Python executes the script. Passive: The script is executed by Python. Active: The engineer fixed the bug. Passive: The bug was fixed by the engineer. Active: AI will transform the industry. Passive: The industry will be transformed by AI.

When to Use Passive Voice

  • When the action is more important than who did it
  • When the agent (doer) is unknown or obvious
  • In technical/scientific writing
  • To maintain formal tone

Common in technical documentation and programming contexts!
